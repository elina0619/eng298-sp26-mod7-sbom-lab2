How does maintaining updated packages illustrate complete mediation?

Maintaining updated packages shows complete mediation because the system is constantly checking and verifying its components instead of assuming they are always safe. Rather than trusting software forever after installation, updates force the system to reevaluate each component and fix any vulnerabilities. This make sures that nothing is overlooked and that every part of the system stays secure.

Why is generating a new SBOM after patching essential to assurance?

Generating a new SBOM after patching is important because it gives an updated picture of what is actually running on the system. It helps confirm whether vulnerabilities were addressed and shows exactly what software is present after updates. Without this, we wouldn’t really know if the patching process made any difference.

What risks persist when organizations delay system updates?

When updates are delayed, vulnerabilities remain in the system, making it easier for attackers to exploit them. This can lead to serious issues like data breaches or unauthorized access. The longer updates are delayed, the more exposed the system becomes, especially as new exploits are discovered over time.

How does this exercise embody the secure design lifecycle principle?

This exercise shows that security isn’t something you do just once, it’s an ongoing process. By scanning for vulnerabilities, applying updates, and checking the results again, it reflects a continuous cycle of improving and maintaining security. This matches the idea of the secure design lifecycle, where systems need to be regularly monitored and updated to stay secure.
